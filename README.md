# Prompt Auditor

A command-line tool that analyzes AI agent prompts for ambiguity issues that cause silent failures, hallucinations, and unpredictable behavior -- then automatically generates a hardened rewrite and verifies it.

## What it does

Writing prompts for AI agents is harder than it looks. Small ambiguities that seem harmless in plain English -- "find the cheapest one," "make the important ones bold," "exclude that title" -- can cause an agent to hallucinate data, silently skip logic, or return wrong results with total confidence.

Prompt Auditor catches those problems before they make it to production.

Given any AI agent prompt, it runs three automated passes:

**Pass 1 -- Audit:** Analyzes the prompt across five failure categories and scores it on a 0–100 confidence scale.

**Pass 2 -- Rewrite:** Generates a hardened version of the prompt that resolves every issue found.

**Pass 3 -- Verify:** Re-runs the full audit on the rewrite to confirm the problems are actually gone and scores it independently.

All three passes are exported to a structured CSV report.

---

## Ambiguity categories detected

| Category | What it catches |
|---|---|
| `SCOPE_LEAKAGE` | Logic from one instruction step bleeding into another and acting on the wrong items |
| `SUBJECTIVE_JARGON` | Vague terms an AI can't calculate without hallucinating (e.g. "the best one", "significant", "important") |
| `LITERAL_TRAPS` | String matching vulnerabilities where casing, punctuation, or naming variants cause silent filter failures |
| `STRUCTURAL_FORMATTING` | Formatting commands written as prose that an AI may print literally instead of applying |
| `SOURCE_DATA_ISSUES` | Missing data sources, logical paradoxes, or context drift in the input the prompt references |

Each finding includes a severity rating (LOW / MEDIUM / HIGH), the exact location in the prompt, an explanation of the risk, and a concrete suggested rewrite.

## Example

**Input prompt:**
```
Show me the cheapest call of duty game you can find.
I just don't want black ops 4 or modern warfare 2.
```

**Sample findings:**
- `[HIGH] SUBJECTIVE_JARGON` -- "cheapest" is undefined without a storefront, pricing type, or data source
- `[HIGH] LITERAL_TRAPS` -- "Modern Warfare 2" matches two separate games (2009 and 2022) and multiple naming variants like "MW2", "MWII", "Modern Warfare II"
- `[MEDIUM] SOURCE_DATA_ISSUES` -- no data source specified; agent has no grounded inventory to query
- `[MEDIUM] SCOPE_LEAKAGE` -- exclusion rule is appended as an afterthought and may be treated as a soft preference rather than a hard filter

**Original score:** 62/100 (MODERATE RISK)

**Rewritten prompt (Pass 2 output):**
```
Search for the Call of Duty game with the lowest current listed price on Steam.
Use only current, live pricing data -- do not rely on historical or estimated prices.

Exclude the following titles entirely, regardless of how they are listed or formatted:
- Call of Duty: Black Ops 4 (also listed as Black Ops IV)
- Call of Duty: Modern Warfare 2 (the 2009 release, also listed as MW2)
- Call of Duty: Modern Warfare II (the 2022 release, also listed as MWII)

If you are unable to retrieve live pricing data, clearly state that and explain
what source or method would be needed to get an accurate result.
Do not guess or estimate prices.
```

**Verification score:** 48/100 (MODERATE RISK) -- residual findings flagged for further hardening

## Setup

**1. Clone the repo**
```bash
git clone https://github.com/your-username/prompt-auditor.git
cd prompt-auditor
```

**2. Install dependencies**
```bash
pip install -r requirements.txt
```

**3. Add your API key**

Copy `.env.example` to `.env` and fill in your Anthropic API key:
```bash
cp .env.example .env
```
Then open `.env` and replace `your-api-key-here` with your key from [console.anthropic.com](https://console.anthropic.com).

---

## Usage

**Audit a prompt passed inline:**
```bash
python prompt_auditor.py --prompt "Summarize the top articles and make the important ones bold."
```

**Audit a prompt from a text file:**
```bash
python prompt_auditor.py --file my_prompt.txt
```

**See all options:**
```bash
python prompt_auditor.py --help
```

Output is printed to the console and saved as a timestamped CSV file (`audit_YYYYMMDD_HHMMSS.csv`) in the current directory.

---

## Scoring

Higher is better. A score of 100 means no issues were detected. A score of 0 means the prompt is critically broken across the board.

Scores are calculated deterministically from the findings -- not generated by the model:

| Score | Grade |
|---|---|
| 80–100 | LOW RISK |
| 50–79 | MODERATE RISK |
| 25–49 | HIGH RISK |
| 0–24 | CRITICAL |

Each finding deducts points based on severity: HIGH costs 12 points (10 severity + 2 flat per-finding penalty), MEDIUM costs 7, LOW costs 4. The flat per-finding penalty ensures that more findings always produce a lower score, even if the severity mix shifts between two runs.
